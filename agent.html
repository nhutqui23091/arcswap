How Auto-Replenish works

A single signature authorizes a Circle Programmable Wallet to act as your agent. The agent then operates within your declared rules — no further interaction needed.

// Step 01
You sign once
An EIP-712 message declares: source chain, target wallet, floor, refill amount, daily cap, expiry. No gas, no on-chain tx — just a signature.
✍
// Step 02
Agent watches
A Circle Programmable Wallet polls your target balance. When it drops below the floor, the trigger fires. Polling adapts to wallet activity — faster when busy, idle otherwise.
👁
// Step 03
Funds move
Agent pulls USDC from your source via EIP-3009 transferAuthorization, deposits to Gateway, and mints to the target chain via CCTP V2 Fast Transfer. Total latency: ~10–20 seconds.
⟶
// Step 04
You stay in control
Hit the daily cap? Agent stops until tomorrow. Want out? Revoke instantly — one click invalidates the signature on-chain. Every execution is logged with tx hashes.
⚑

Built on Circle's Agent Stack

◈
Programmable Wallets
MPC-managed wallets with built-in policy engine — your agent's identity.
◆
Circle Gateway
Unified balance abstraction — issues attestations for near-instant cross-chain mint.
◇
CCTP V2 Fast Transfer
8–20 second native USDC burn-and-mint between supported chains.
⌬
EIP-3009 / EIP-2612
Off-chain signed authorizations — no upfront approve tx, no gas on source chain.
⬡
Arc L1
USDC-native gas, sub-second blocks — purpose-built settlement for agentic flows.
▴
x402 (roadmap)
HTTP 402 payment negotiation — let your agent pay any x402-aware service autonomously.
